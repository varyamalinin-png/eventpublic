# Исправление ошибки 403 Forbidden в Yandex Cloud Email API

## Проблема
Ошибка `403 Forbidden` при отправке email через Yandex Cloud Postbox API.

## Возможные причины:

1. **Домен не подтвержден в Yandex Cloud Postbox**
   - Домен `iventapp.ru` должен быть добавлен и подтвержден в консоли Yandex Cloud
   - Нужно добавить DNS записи для подтверждения домена

2. **IAM токен истек или недействителен**
   - IAM токены действительны ограниченное время
   - Нужно обновить токен

3. **Сервисный аккаунт не имеет прав**
   - Сервисный аккаунт должен иметь роль `postbox.editor` или `postbox.admin`

4. **Email API не активирован**
   - В консоли Yandex Cloud нужно активировать Email API для домена

## Решение:

### 1. Проверьте домен в консоли Yandex Cloud:
- Перейдите в раздел "Email API" или "Postbox"
- Убедитесь, что домен `iventapp.ru` добавлен и подтвержден
- Проверьте, что DNS записи добавлены правильно

### 2. Обновите IAM токен:
```bash
# На локальной машине
cd server
node scripts/get-yandex-iam-token.js
# Скопируйте новый токен и обновите в .env на сервере
```

### 3. Проверьте права сервисного аккаунта:
- В консоли Yandex Cloud найдите сервисный аккаунт
- Убедитесь, что у него есть роль `postbox.editor` или `postbox.admin`

### 4. Активируйте Email API:
- В консоли Yandex Cloud перейдите в раздел Email API
- Убедитесь, что сервис активирован для вашего облака
