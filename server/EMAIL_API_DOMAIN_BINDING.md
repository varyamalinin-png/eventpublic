# Связывание сервисного аккаунта с доменом в Yandex Cloud Email API

## Проблема
Ошибка 403 Forbidden даже после назначения роли `postbox.editor` на каталог и облако.

## Решение через веб-консоль:

1. Откройте консоль Yandex Cloud: https://console.yandex.cloud
2. Перейдите в раздел **Email API** (или **Postbox**)
3. Откройте домен `iventapp.ru` (ID: `egtjmmg0b2fvtcvvsdj3`)
4. Найдите раздел **"Доступ"**, **"Service Accounts"** или **"Права доступа"**
5. Добавьте сервисный аккаунт:
   - Имя: `email-api-sender`
   - ID: `aje9ll01mrc2e82m5cv8`
   - Роль: `postbox.editor` или `postbox.admin`

## Альтернатива - через API:

Если в веб-консоли нет такой возможности, возможно нужно использовать другой endpoint или способ авторизации.

## Проверка:

После связывания попробуйте отправить письмо снова. Если ошибка сохраняется, возможно нужно:
- Обратиться в поддержку Yandex Cloud
- Проверить, что Email API полностью активирован для вашего облака
