# Финальный анализ ошибки 403 Forbidden в Yandex Cloud Email API

## Текущая ситуация:
- ✅ Домен `iventapp.ru` подтвержден (Status: Active, Verification: Success)
- ✅ Сервисный аккаунт `email-api-sender` создан (ID: `aje9ll01mrc2e82m5cv8`)
- ✅ Роли назначены:
  - `postbox.editor` на каталог и облако
  - `postbox.admin` на каталог и облако
  - `postbox.sender` на каталог и облако
- ✅ IAM токен генерируется и обновляется
- ❌ Все равно получаем `403 Forbidden`

## Возможные причины:

1. **Домен не связан с сервисным аккаунтом**
   - В веб-консоли нет возможности связать сервисный аккаунт с доменом
   - Возможно, нужно использовать другой способ

2. **Домен не активирован для отправки**
   - Домен подтвержден, но может быть не активирован для отправки писем
   - Нужно проверить в веб-консоли Email API

3. **Неправильный endpoint или формат запроса**
   - Используем `https://postbox.cloud.yandex.net/v2/email/outbound-emails`
   - Может быть нужен другой endpoint или формат

4. **Проблема с IAM токеном**
   - Токен генерируется правильно
   - Но может быть проблема с правами, которые он содержит

## Рекомендации:

1. **Проверить в веб-консоли Email API:**
   - Открыть домен `iventapp.ru`
   - Проверить, есть ли раздел "Отправка" или "Sending"
   - Убедиться, что домен активирован для отправки

2. **Обратиться в поддержку Yandex Cloud:**
   - Это может быть известная проблема
   - Поддержка может помочь связать сервисный аккаунт с доменом

3. **Попробовать использовать API ключ вместо IAM токена:**
   - Возможно, для Email API нужен API ключ, а не IAM токен

4. **Проверить документацию на наличие дополнительных шагов:**
   - Может быть нужно выполнить дополнительные действия для активации домена
