# Решение ошибки 403 Forbidden в Yandex Cloud Email API

## Проблема
Ошибка `403 Forbidden` при отправке email через Yandex Cloud Postbox API, несмотря на:
- ✅ Домен `iventapp.ru` подтвержден (Status: Active, Verification: Success)
- ✅ Сервисный аккаунт `email-api-sender` имеет роль `postbox.editor` на каталоге

## Возможные причины:

1. **Сервисный аккаунт не связан с доменом в Email API**
   - В консоли Yandex Cloud нужно явно связать сервисный аккаунт с доменом

2. **Роль нужна на уровне облака, а не каталога**
   - Попробуйте назначить роль `postbox.editor` на уровне облака

3. **Домен не активирован для отправки**
   - Проверьте, что домен полностью настроен и активирован

## Решение через веб-консоль:

1. Откройте консоль Yandex Cloud: https://console.yandex.cloud
2. Перейдите в раздел **Email API** (Postbox)
3. Откройте домен `iventapp.ru`
4. Найдите раздел **"Доступ"** или **"Service Accounts"**
5. Добавьте сервисный аккаунт `email-api-sender` (ID: `aje9ll01mrc2e82m5cv8`)
6. Убедитесь, что у него есть права на отправку email

## Альтернативное решение:

Если в веб-консоли нет возможности связать сервисный аккаунт с доменом, возможно нужно:
- Использовать другой способ авторизации
- Проверить, что Email API активирован для вашего облака
- Обратиться в поддержку Yandex Cloud
